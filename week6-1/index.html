    User Testing

    By testing our low-fi wireframes, we were able to identify some weak areas of our design before proceeding. This phase is especially important as we will skipping the mid-fi step and going straight to the hi-fi mockup.

    This article will cover how we tested,the main issues with our current design, and how we plan to fix them

    Testing

    We asked people to try and complete our user scenarios using the paper wireframes. Ideally, these wireframes would be configured in Figma using the prototype tools, but due to time constraints in the class schedule we stuck with paper versions.

    The testers would 'click' throughout the page as they attempted to solve the user scenario, like find allergy information or complete a bulk order. This helped us to see the flaws in our design and some issues users might run into.

    Issues

    Order Menu Item

    One user story required testers to view their existing order for information on pickup, price, items, etc. We found that in naming the menu item 'Order' confused many testers, making them think that the menu item was to place a new order, not check the details of an existing one.

    We will eliminate this confusion by renaming this menu item to be 'My Orders'.

    Box Size Menu

    A crucial part of the ordering process is to first select the amount of cupcakes you want to order, as Toronto Cupcake only sells in set amounts and the price changes the more you buy.

    Our first design showed the box size menu option at the top of the page, consisting of four options: 6 pcs, 12 pcs, 24 pcs, and Bulk. All testers skipped over this step and went straight to clicking on cupcakes.

    We can fix this issue by making it more clear that there are steps to ordering. By adding numbered steps to each of the three headers on that page, Box Size, Cupcakes, and Order, we can help users to identify which steps they need to complete before moving on.

    With these issues identified and solved, we can move on to planning our Hi-Fi. The next article covers the rebrand of Toronto Cupcakes.
